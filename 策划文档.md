## 二、版本历史

### v1.0 → v2.0 主要更新（2026-04-20 下午）
- 新增用户系统（注册/登录）
- 新增云端排行榜
- 新增个人记录（最近20次）
- 优化难度数值
- 输了也能保存分数

### v2.0 → v2.1 主要更新（2026-04-20 晚）
- 新增技能系统：锁定打击（J键）+ 清屏弹（K键）
- 按键重新设计
- 预留子弹类型扩展（散弹、穿透弹）

### v2.1 → v2.2 主要更新（2026-04-20 深夜）
- **射击系统**：保持自动瞄准（±5°偏差），不做方向射击
- **能量系统**：擦弹充能 → **击杀敌人充能**
- **J技能**：改为**激光锁定**，持续伤害，自动追踪下一个目标
- **K技能**：改为**水波扩散**，固定500伤害秒杀小怪
- **血条**：敌人身体即血条（饼图/部分填充形式）
- **Boss击中充能**：击中Boss获得部分能量，Boss击杀充满
- **伤害跳字**：命中敌人显示浮动伤害数字
- **帧率固定**：高刷新率屏幕下游戏速度保持一致

### v2.2 → v2.3 主要更新（2026-04-21 上午）
- **玩家朝向**：默认朝上，**松开按键后保持最后朝向**
- **敌人死亡子弹**：圆形死亡不发射子弹；其他敌人死亡发射子弹并朝向玩家（square=1发, hexagon=2发, chaser=4发）
- **K技能触发**：改为1格能量（100）即可触发（原2格=200）
- **敌人移动**：限制在屏幕范围内，增加向玩家追踪强度（22%）
- **Chaser敌人**：新增追踪型敌人，会冲刺撞击玩家
- **难度下调**：减少各波次敌人数量和弹幕密度

## 三、视觉风格

### 美术基调

- **风格**：几何抽象 + 霓虹光效
- **背景**：深色宇宙星空（黑/深蓝底色）
- **所有角色和敌人**：纯几何图形构成

### 几何角色设计

| 角色 | 形状 | 颜色 | 特点 |
|------|------|------|------|
| 玩家 | 三角形 | 青色 | 默认朝上，松开按键保持朝向 |
| 敌人-1 | 圆形 | 红色 | 最小型敌人，身体即血条，死亡不发射子弹 |
| 敌人-2 | 方形 | 橙色 | 中型敌人，身体即血条，死亡发射1发朝向玩家的子弹 |
| 敌人-3 | 六边形 | 紫色 | 大型敌人，身体即血条，死亡发射2发朝向玩家的子弹 |
| 敌人-4 | 菱形 | 粉色 | 追踪型敌人，会冲刺撞击玩家，死亡发射4发朝向玩家的子弹 |
| BOSS | 八边形 + 外围光圈 | 粉红 | 每阶段形态变化 |

### 子弹设计

| 子弹类型 | 形状 | 颜色 | 说明 |
|---------|------|------|------|
| 玩家子弹 | 小圆形 | 青色 | 默认 |
| 敌人子弹 | 椭圆形 | 红色/橙色 | 敌方弹幕 |

### 特效

- 敌人击破：几何碎片爆炸
- J技能激光：黄色光束 + 命中粒子
- K技能水波：从玩家位置向外扩散的圆环
- 伤害跳字：命中敌人时飘出黄色数字
- 玩家受伤：屏幕闪红

## 四、核心玩法

### 4.1 玩家操作

| 操作 | 按键 |
|------|------|
| 移动 | WASD / 方向键（8方向，自动朝向） |
| J技能（激光锁定） | J（按住，消耗能量） |
| K技能（水波清屏） | K（能量≥1格时释放） |
| 暂停 | ESC / P |

### 4.2 射击系统

**自动瞄准规则**：
- 子弹自动朝**最近敌人**发射
- 有轻微偏差（±5°），让手感更自然
- 玩家专注于走位和释放技能

**子弹属性**：
- 速度：12 px/帧
- 半径：6px
- 伤害：15

### 4.3 能量系统

| 属性 | 数值 |
|------|------|
| 能量上限 | 200（2格） |
| 1格能量 | 100 |
| 充能方式 | **击杀敌人** |
| 击杀敌人充能 | +25 能量 |
| Boss击中充能 | +12.5 能量 |
| Boss击杀充能 | **充满（200）** |
| J技能消耗 | 100能量/秒（长按持续消耗到0） |

### 4.4 技能系统

#### 技能1：激光锁定（J键按住）

| 属性 | 数值 |
|------|------|
| 触发条件 | 能量 > 0（1格=100可触发） |
| 终止条件 | 能量耗尽到0 |
| 消耗速度 | 100能量/秒（按住持续消耗） |
| 效果 | 发射黄色激光束，持续伤害锁定目标 |
| 激光伤害 | 2点HP/帧 |
| 切换目标 | 目标死亡后自动切换到下一个最近敌人 |
| 加移速 | +2 px/帧 |
| 视觉 | 玩家到目标之间有黄色光束 + 命中粒子 |

**策略深度**：
- 选择锁谁 = 决定先杀哪个
- 能量规划：长按耗尽 vs 短按保留
- 目标死亡后自动切换，持续压制

#### 技能2：水波清屏（K键）

| 属性 | 数值 |
|------|------|
| 触发条件 | 能量 ≥ 100（1格即可） |
| 效果 | 从玩家位置向外扩散水波 |
| 固定伤害 | 500（HP<500的小怪直接秒杀） |
| 对Boss伤害 | 500 |
| 视觉 | 青色圆环从玩家向外扩散直到全屏 |

**策略深度**：
- 留着不用浪费（满能量不放等于损失）
- 等敌人密集时放 = 最大化收益
- 危急时刻保命用

### 4.5 血条设计

**核心概念**：敌人身体即血条

| 敌人 | 血条表现 |
|------|---------|
| 圆形 | 饼图形式，HP剩多少%显示多少%实心弧，缺失部分是半透明轮廓 |
| 方形 | 从左边开始填充，HP剩多少%就填充多少%宽度 |
| 六边形 | 顺时针填充，HP剩多少%就填充多少个角 |

### 4.6 策略深度总结

| 决策点 | 选项 | 权衡 |
|-------|------|------|
| 走位 | 躲避弹幕 | 移动决定生存 |
| J技能 | 长按耗尽 vs 短按保留 | 爆发力 vs 续航 |
| J技能目标 | 优先杀谁 | 影响清场速度 |
| K技能时机 | 敌人密集时 vs 危急保命 | 最大化收益 vs 救命 |

## 五、敌人与波次

### 5.1 波次设计（共5波 + BOSS）

| 波次 | 敌人构成 | 持续时间 |
|------|---------|---------|
| 第1波 | 10个圆形（红） | 20秒 |
| 第2波 | 8个圆形 + 4个方形（橙） | 25秒 |
| 第3波 | 6个圆形 + 5个方形 + 8个追踪者 + 2个六边形 | 30秒 |
| 第4波 | 6个圆形 + 6个方形 + 10个追踪者 + 4个六边形 | 35秒 |
| 第5波 | 8个方形 + 12个追踪者 + 5个六边形 | 40秒 |
| BOSS | 1个BOSS（血量：3000） | 无限时间 |

### 5.2 敌人移动模式

| 敌人 | 移动模式 |
|------|---------|
| 圆形（红） | 从屏幕上方进入，直线向下移动，限制在屏幕上半区 |
| 方形（橙） | 从左侧进入，之字形移动，限制在屏幕内 |
| 六边形（紫） | 从右侧进入，弧形移动，限制在屏幕内 |
| 追踪者（粉） | 从右侧进入，追踪玩家移动，会冲刺撞击 |
| BOSS | 沿屏幕边缘移动 |

### 5.3 敌人子弹预警机制

**所有敌人在发射子弹前有预警**：

| 敌人 | 预警方式 | 预警时间 |
|------|---------|---------|
| 圆形 | 敌人闪烁1次 | 10帧（约0.16秒） |
| 方形 | 敌人闪烁2次 | 15帧（约0.25秒） |
| 六边形 | 敌人发光膨胀 | 20帧（约0.33秒） |
| 追踪者 | 冲刺前发光警告 | 30帧（约0.5秒） |
| BOSS | 屏幕边缘出现光圈 + BOSS发光 | 30帧（约0.5秒） |

### 5.4 敌人死亡子弹机制

| 敌人 | 死亡发射子弹数 | 子弹行为 |
|------|---------------|---------|
| 圆形 | 0（不发射） | - |
| 方形 | 1发 | 朝向玩家 |
| 六边形 | 2发 | 朝向玩家，带扩散 |
| 追踪者 | 4发 | 朝向玩家，带扩散 |

### 5.5 BOSS设计

**形态切换视觉区分**：

| 形态 | HP范围 | 颜色变化 | 移动变化 | 特效 |
|------|--------|---------|---------|------|
| 形态1 | 100%-60% | 粉红色 | 缓慢沿边移动 | 正常发光 |
| 形态2 | 60%-30% | 橙红色 | 快速移动 | 闪烁加剧 |
| 形态3 | 30%-0% | 深红紫色 | 原地不动 | 强烈脉动光效 |

### 5.6 BOSS弹幕花样

**形态1（100%-60%血）**：
- 缓慢移动
- 发射12方向圆形弹幕（全方向散射）
- 发射间隔：72帧（1.2秒）

**形态2（60%-30%血）**：
- 快速移动
- 发射追踪弹（会缓慢改变方向）
- 发射间隔：50帧（0.83秒）

**形态3（30%-0%血）**：
- 原地不动
- 同时发射圆形弹幕 + 追踪弹
- 发射间隔：36帧（0.6秒）

## 六、计分系统

### 6.1 通关时计分

| 得分来源 | 分数 |
|---------|------|
| 击破敌人 | 100 × 敌人等级 |
| 速度奖励 | 3000 - 通关秒数 × 20 |
| 无伤奖励 | +500 |
| 生命奖励 | 剩余命 × 100 |

**综合分 = 击破分 + 速度分 + 无伤奖励 + 生命奖励**

### 6.2 游戏结束时计分（输了）

| 得分来源 | 分数 |
|---------|------|
| 击破敌人 | 100 × 敌人等级 |
| 无伤奖励 | +500（如全程无伤） |

**不享受速度奖励和生命奖励**

### 6.3 评价等级

| 等级 | 条件 |
|------|------|
| **S** | 综合分 ≥ 5000 |
| **A** | 综合分 ≥ 3500 |
| **B** | 综合分 ≥ 2000 |
| **C** | 综合分 ≥ 1000 |
| **D** | 综合分 < 1000 |

## 七、技术方案

### 技术栈

| 项目 | 选择 |
|------|------|
| 前端渲染 | HTML5 Canvas |
| 前端语言 | 原生 JavaScript |
| 后端 | Node.js + Express |
| 数据库 | SQLite3 |
| 认证 | JWT + bcrypt |
| 打包 | 前端单HTML文件 |
| 部署 | Nginx反向代理 |

### 帧率处理

- 使用固定时间步长（Fixed Timestep）确保60fps
- 高刷新率屏幕（144Hz/240Hz）下游戏速度保持一致
- 避免帧率依赖的物理计算

### 架构

```
用户浏览器
    ↓ HTTPS
Nginx (xiaoqicoding.com)
    ├── /game/geo-storm/ → 静态游戏页面
    └── /api/geostorm/ → 反向代理到 3001端口
                            ↓
                      Node.js 后端
                            ↓
                      SQLite3 数据库
```

### API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/geostorm/auth/register` | POST | 用户注册 |
| `/api/geostorm/auth/login` | POST | 用户登录 |
| `/api/geostorm/auth/verify` | GET | 验证Token |
| `/api/geostorm/scores` | POST | 保存分数（需登录） |
| `/api/geostorm/scores/me` | GET | 我的最近20次记录（需登录） |
| `/api/geostorm/leaderboard` | GET | 总榜单TOP20（无需登录） |

## 八、用户系统流程

### 8.1 游戏流程

```
打开游戏
    ↓
登录/注册界面（必须登录才能玩）
    ↓
主菜单
├── 开始游戏
├── 我的记录（最近20次）
└── 总榜单（TOP 20）
    ↓
游戏结束/通关
    ↓
自动保存成绩到云端
    ↓
显示成绩（5秒后返回主菜单）
```

### 8.2 用户必须登录

- 未登录用户无法开始游戏
- 登录后游戏成绩自动保存到云端
- 每个用户可查看自己的最近20次记录
- 所有用户共享总榜单TOP 20
